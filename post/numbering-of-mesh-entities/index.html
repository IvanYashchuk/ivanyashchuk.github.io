About numbering of mesh entities

Jul 12, 2017 00:00 · 426 words · 2 minutes read

The topological entities of a cell (or mesh) are referred to as mesh entities. A mesh entity can be identified by a pair $(d, i)$, where $d$ is the topological dimension of the mesh entity and $i$ is a unique index of the mesh entity. Mesh entities are numbered within each topological dimension from $0$ to $n_d − 1$, where $n_d$ is the number of mesh entities of topological dimension $d$. Mesh entity numbering should be the same across all software components (FIAT, FFC, DOLFIN).

Quadrilateral and hexahedron cells are constructed as FIAT.reference_element.TensorProductCell of interval cells.

from fenics import *
from ffc.fiatinterface import create_element
# Create UFL's P1 Finite Element on interval
ufl_interval = FiniteElement("P", interval, 1)

# Create FIAT's Finite Element on interval
fiat_interval = create_element(ufl_interval)

Now fiat_interval is instance of FIAT.Lagrange on reference element UFCInterval

In [..]: fiat_interval.get_reference_element()
Out[..]: <FIAT.reference_element.UFCInterval at 0x7fdb4aa72da0>

The topological entities of a cell can be accessed with reference_element.get_topology()

In [..]: fiat_interval.ref_el.get_topology()
Out[..]: {0: {0: (0,), 1: (1,)}, 1: {0: (0, 1)}}

Topology of a cell is stored as dict of dicts, mesh entity identified by a pair $(d, i)$ can accessed by index [d][i]. Entities of topological dimension 0 here are vertices, entities of dimension 1 (codimension zero) is the cell itself.

Quadrilateral is TensorProductCell(UFCInterval(), UFCInterval())

from FIAT.reference_element import TensorProductCell, UFCInterval
quad_cell = TensorProductCell(UFCInterval(), UFCInterval())

In [..]: quad_cell.get_topology()
Out[..]:
{(0, 0): {0: (0,), 1: (1,), 2: (2,), 3: (3,)},
 (0, 1): {0: (0, 1), 1: (2, 3)},
 (1, 0): {0: (0, 2), 1: (1, 3)},
 (1, 1): {0: (0, 1, 2, 3)}}

Now topological dimension is stored as the tuple of dimensions in different “directions”, it comes from the tensor product operation. The topological dimension $d$ is the sum of the elements in the tuple. It is useful to store the dimension as a tuple if interval elements of different order are used in different directions. Usually the order is the same.

It is more practical to store the dimension as an integer. Therefore new FIAT’s reference cell were introduced UFCQuadrilateral and UFCHexahedron. The entity dimensions of these new elements are “flattened” to normal elements.

from FIAT.reference_element import UFCQuadrilateral
quad_cell = UFCQuadrilateral()

In [..]: quad_cell.get_topology() Out[..]: {0: {0: (0,), 1: (1,), 2: (2,), 3: (3,)}, 1: {0: (0, 1), 1: (2, 3), 2: (0, 2), 3: (1, 3)}, 2: {0: (0, 1, 2, 3)}} `` The tensor product cell can be accessed if needed byquad_cell.product`.

Mesh entity description for quadrilateral and hexahedron mesh in FFC and DOLFIN was modified to have the same topology as FIAT’s flattened UFCQuadrilateral and UFCHexahedron.
